# Location Stock Indicator：Lite / Pro プラン設計

**目的**: 公開アプリとして Location Stock Indicator を提供する際の **Lite / Pro** プラン構成・料金・機能制御を整理する。  
**参照**: POS Stock（stock-transfer-pos）の `docs/PUBLIC_APP_PLAN_FEATURES_DESIGN.md`、`docs/ADMIN_HOME_AND_PLAN_UI_REQUIREMENTS.md`。

---

## 1. プラン概要

| 項目 | Lite | Pro |
|------|------|-----|
| **料金** | **$20/月**（定額） | **$20/月 ＋ ロケーション数 × $2**（従量課金） |
| **ロケーション数** | **10ロケーションまで**（上限あり） | **制限なし**（ロケーション指定なし） |
| **エリア設定** | なし | あり（全機能） |
| **近隣店舗表示** | なし | あり |
| **店舗受け取り（ボタン・モーダル等）** | なし | あり |
| **分析（管理画面）** | なし | あり（将来実装時） |
| **その他** | 基本の在庫表示・並び順・閾値・公開名等は利用可能 | 上記に加え全ての機能を利用可能 |

- **Lite**: シンプルな「ロケーション別在庫表示」に限定。エリア・近隣店舗・店舗受け取り・分析は利用できない。
- **Pro**: ロケーション数に制限なく、エリア・近隣店舗・店舗受け取り・分析を含む全機能を利用可能。料金は $20 に加え、ロケーション数に応じて $2/ロケーションで従量課金。

---

## 2. 料金の整理

### 2.1 Lite

| 項目 | 内容 |
|------|------|
| **月額** | $20（定額） |
| **ロケーション** | 10ロケーションまで。11ロケーション以上はプラン不一致として「Pro への変更」を案内する想定。 |
| **課金形態** | Recurring Application Charge（月額固定）のみ。Managed pricing で「Lite - 10 locations」等のプラン名で設定。 |

### 2.2 Pro

| 項目 | 内容 |
|------|------|
| **基本料金** | $20/月 |
| **従量** | 1ロケーションあたり $2/月（ロケーション数に上限なし） |
| **計算式** | **$20 ＋ （ロケーション数 × $2）**  
  例: 5ロケーション → $20 + $10 = $30/月、20ロケーション → $20 + $40 = $60/月 |
| **課金形態** | Recurring（$20 基本料）＋ **Usage-based billing**（ロケーション数 × $2）。Shopify の App Subscription API（Recurring ＋ Usage line item）で実装。 |

## 3. 機能とプランの対応

### 3.1 管理画面・テーマで使う機能の対応表

| 機能 | 設定場所・経路 | Lite | Pro |
|------|----------------|------|-----|
| 閾値（在庫なし/残りわずか/在庫あり） | 在庫表示設定 → config | ○ | ○ |
| 在庫マーク・ラベル・メッセージ・注意書き | 在庫表示設定 | ○ | ○ |
| 並び順・上部固定ロケーション | ロケーション設定 | ○ | ○ |
| ロケーション表示ON/OFF・公開名・リンクURL | ロケーション設定 | ○ | ○ |
| **エリア設定**（グルーピング・折りたたみ・未設定の見出し・グループ一覧） | ロケーション設定 | **×** | ○ |
| **近隣店舗表示**（近隣店舗を表示・その他ロケーションの見出し） | ロケーション設定 | **×** | ○ |
| **店舗受け取り**（ボタン表示・ラベル・リダイレクト・文言・モーダル） | ロケーション設定 | **×** | ○ |
| **分析（管理画面）**（エリア表示回数・近隣クリック・店舗受け取りクリック等） | 将来：分析画面 | **×** | ○ |

- **Lite** では「エリア」「近隣店舗」「店舗受け取り」「分析」に該当する設定・表示・API を無効化する（管理画面では非表示または無効、テーマのスニペットでは config に応じて表示しない）。

### 3.2 実装時の制御方針（案）

- **プラン取得**: 公開アプリ時のみ、`currentAppInstallation.activeSubscriptions` 等からプラン名を取得し、`lite` / `pro` を判定（POS Stock の `getPlanFromActiveSubscriptions` と同様）。
- **Lite のとき**:
  - 管理画面のロケーション設定で「エリア設定」「近隣店舗表示設定」「店舗受取設定」のセクションを非表示または「Pro で利用可能」案内に差し替え。
  - メタフィールドの `config.future` で `groupByRegion` / `regionAccordionEnabled` / `nearbyFirstEnabled` / `showOrderPickButton` 等を **保存・適用しない**（または上書きで OFF に固定）。App Proxy は config をそのまま返してよく、スニペット側で「エリア・近隣・店舗受け取り」を使わないようにする場合は、アプリから渡すフラグで制御するか、Lite ではそれらのキーを送らないようにする。
  - 分析画面（将来）は非表示または「Pro でご利用いただけます」にリダイレクト。
- **Pro のとき**: 現状どおり全機能を利用可能。ロケーション数制限なし。
- **ロケーション数と Lite の制限**: 実際のロケーション数が 10 を超えている場合は「プラン不一致」として、料金プラン画面へ誘導し、Pro への変更を案内する（POS Stock の `locationPlanMismatch` と同様の考え方）。

## 4. カスタムアプリ・公開アプリ・開発ストアの切り替え（POS Stock 同様）

| 対象 | 判定方法 | プラン制限 | 機能 |
|------|----------|------------|------|
| **カスタムアプリ**（自社用） | 環境変数 `APP_DISTRIBUTION=inhouse` | なし | **全機能**（Pro 相当） |
| **開発ストア** | `shop.plan.partnerDevelopment === true` 等 | なし | **全機能**（課金せず Pro 相当） |
| **公開アプリ（本番ストア）** | `APP_DISTRIBUTION=public`（または未設定） | あり | Lite / Pro に応じて制限 |

- **カスタムアプリ**: デプロイ先（例: 自社用 Render）に `APP_DISTRIBUTION=inhouse` を設定。プラン判定をスキップし、常に全機能・ロケーション数制限なし。
- **開発ストア**: パートナーが作成した開発用ストアでは課金しない。アプリ内では **全機能を解放**（Pro 相当）として扱い、管理画面・テーマとも制限をかけない。POS Stock と同様に `shop.plan.partnerDevelopment` で開発ストアを判定する。
- **公開アプリ（本番）**: 上記以外のストアでは Lite / Pro の料金・機能制御を適用する。
- 同一コードベースで、デプロイ先・環境変数・ショップ情報により振る舞いを切り替える（POS Stock と同様）。

## 5. 管理画面の出し分け：今の構成のままでよいか

### 5.1 結論：**今のままで問題ない**（拡張設定にまとめなくてよい）

- **現状**: ロケーション設定画面では、**エリア設定**・**近隣店舗表示設定**・**店舗受取設定**がすでに**別々のカード／セクション**になっている。
- **出し分け方針**: プランが Lite（かつ公開アプリ）のとき、この 3 セクションを**それぞれ**「非表示」または「Pro で利用可能」の案内に差し替えればよい。わざわざ「拡張設定」という 1 つのまとまりにしなくても、既存のセクション単位でプラン判定をかけるだけで実装できる。
- **拡張設定にまとめる場合**: 「Pro で利用可能な機能」を 1 つの折りたたみ（例: 「拡張設定（Pro）」）にまとめると、Lite のときは**そのブロックごと非表示**にでき、条件分岐が 1 箇所で済む。ただし、Pro ユーザーにとっては「今までバラバラに見えていたセクションが 1 つにまとまる」という UI 変更になる。現状の並び・見た目を変えたくないなら、**今のまま各セクションでプラン判定**がおすすめ。
- **推奨**: **今の構成のまま**、エリア設定・近隣店舗表示設定・店舗受取設定の**各セクション**で「公開アプリかつ Lite なら非表示（または Pro 案内カードを表示）」とする。コードの変更も少なく、Pro ユーザーの操作感も現状のまま維持できる。

### 5.2 管理画面・スキーマ（テーマ）のプラン出し分けの整理

| 対象 | 出し分けのやり方 |
|------|------------------|
| **管理画面** | loader でプラン（と distribution・開発ストア）を取得。Lite のときは「エリア設定」「近隣店舗表示設定」「店舗受取設定」の 3 セクションを非表示、または 1 枚の案内カード（「Pro でご利用いただけます」＋料金プランへのリンク）に差し替え。分析画面（将来）は Pro のみ表示。 |
| **保存・config** | Lite のときは上記 3 セクションのフォームを出さないため、既存の config（メタフィールド）の該当キーは更新しない。既に Pro から Lite にダウングレードした場合は、App Proxy 側で「プランが Lite なら groupByRegion / nearbyFirstEnabled / showOrderPickButton 等を強制 OFF で返す」などのフォールバックを検討可能。 |
| **テーマ（スニペット）** | App Proxy が返す `config`（と必要なら `plan` や `features`）に従い、エリア・近隣店舗・店舗受け取りの表示を出し分ける。Lite ではこれらのフラグを OFF にして返すため、スニペットは既存の `config.future` の解釈のままでよい（フラグがなければ表示しない）。 |
| **テーマ（スキーマ）** | ブロックのスキーマ（カスタマイザーの項目）は**全ショップ共通**で同じ。エリア／近隣店舗／店舗受け取りの「デザイン」項目は Pro でしか使われないが、Lite では管理画面でそれらの機能を OFF にしているため、テーマ側では表示されない。スキーマ項目をプランで非表示にする必要はなく、**config で制御するだけでよい**。 |

- **まとめ**: 管理画面は**今のセクション構成のまま**、各セクションをプランで表示／非表示（または Pro 案内）にすればよい。テーマは config に任せ、スキーマは共通のままでよい。

### 5.3 テーマのスキーマ（ブロックスキーマ）はプランで出し分けできるか

**結論: スキーマ項目そのものをプランで非表示にすることはできません。**

- **理由**: Theme App Extension のブロックで定義する `{% schema %}` は、テーマエディタ（カスタマイザー）に表示する項目を**静的に**定義するものです。アプリのサーバーから「このストアは Lite だからこの項目を隠す」のように動的にスキーマを変える API は Shopify にありません。
- **取りうる対応**:
  - **config で制御する**: 管理画面でプランに応じて保存する内容を変え（Lite のときはエリア・近隣・店舗受け取りを OFF）、App Proxy が返す `config` にそれらのフラグを含めない（または OFF で返す）。スニペット側では「フラグがなければ表示しない」という既存の解釈のままで、結果として商品ページ上の表示がプランで出し分けされる。
  - **スキーマは共通のまま**: カスタマイザーの項目（色・余白・ボタン種類など）は全プランで同じ項目が並んでいてよい。実際に「効く」かどうかは config 次第（例: エリアの見出し色はスキーマで設定できるが、Lite ではエリア機能自体が OFF なので見出しは表示されない）。
- **まとめ**: プランによる出し分けは**管理画面の表示・保存**と **App Proxy が返す config** で行い、テーマのスキーマ項目をプランで増減することはしない。

## 6. 管理画面・料金プランページの構成（案）

- **ホーム**: 現在のプラン（Lite / Pro）・ロケーション数・導入ステップを表示。Lite のときは「エリア・近隣・店舗受け取り・分析は Pro で利用可能」の案内を表示可能。
- **料金プランページ**（`/app/plan`）:  
  - Lite: $20/月、10ロケーションまで、機能は基本の在庫表示に限定。  
  - Pro: $20 + $2/ロケーション、ロケーション数無制限、全機能。  
  - 「このプランを選択する」→ Shopify のプラン選択（Managed pricing）または Billing API の confirmationUrl へ遷移。
- **ロケーション設定**: Pro 専用セクションのタイトルは「エリア/近隣店舗のデザイン（Proプラン）」「近隣店舗表示設定（Proプラン）」「店舗受け取りボタン（Proプラン）」とし、タイトル横に Pro バッジを表示。Lite のときは同セクションを表示したままグレーアウト・編集不可とする。

## 開発・検証用：開発ストアで Lite を強制する

開発ストアでは通常「全機能（Pro 相当）」になるが、**Lite プランの挙動を確認したい**ときは環境変数でプランを上書きできる。

| 環境変数 | 値 | 動作 |
|----------|-----|------|
| **FORCE_PLAN_LITE** | `1` | プランを **Lite** に強制する。開発ストアや本番ストアで、管理画面・App Proxy が Lite として動作する（Pro セクションのグレーアウト、分析の制限、エリア・近隣除外列の編集不可など）。 |

**使い方**

1. ローカルまたは Render の環境変数に `FORCE_PLAN_LITE=1` を設定する。
2. アプリを開き、ホームの「現在の料金プラン」が Lite、ロケーション設定で Pro セクションがグレーアウト、分析で「Pro でご利用いただけます」になることを確認する。
3. 確認後は **必ず `FORCE_PLAN_LITE` を削除または未設定に戻す**（本番で Lite を強制しないため）。
