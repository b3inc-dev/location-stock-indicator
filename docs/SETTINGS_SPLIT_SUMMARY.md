# 設定の振り分けまとめ（在庫表示設定 vs カスタマイザー）

実装前に、「管理画面の在庫表示設定にまとめるもの」と「カスタマイザーで見て調整するもの」をはっきり分けた整理です。

---

## 方針

| 置き場所 | 役割 | 判断の目安 |
|----------|------|------------|
| **在庫表示設定（管理画面）** | 「何を・どう表示するか」の**ルールと文言**を**全ブロックで共通**にしたいもの | ショップ全体で統一したい。文言・閾値・表示内容のルール。 |
| **カスタマイザー（テーマブロック）** | **見た目・レイアウト**を、プレビューを見ながらブロックごとに調整したいもの | 文字サイズ・間隔・色・配置など「見て決めたい」項目。 |

---

## 1. 在庫表示設定（管理画面）にまとめるもの

「在庫の意味・表示ルール・文言」を一元管理する項目。現在の `app.settings.jsx` にあるものに加え、テーマから移す候補も含めて一覧にしています。

### 1.1 すでに在庫表示設定にあるもの

| 項目 | config のキー（例） | 説明 |
|------|---------------------|------|
| **閾値** | `thresholds.outOfStockMax`, `inStockMin` | 在庫なし／在庫ありの境界値 |
| **在庫マークの文字** | `symbols.inStock`, `lowStock`, `outOfStock` | ◯△✕ など（顔文字・絵文字も可） |
| **在庫数ラベル・前後の文字** | `quantity.quantityLabel`, `wrapperBefore`, `wrapperAfter` | 「在庫」や ( ) など |
| **ステータスラベル** | `labels.inStock`, `lowStock`, `outOfStock` | 「在庫あり」「残りわずか」「在庫なし」 |
| **メッセージ文言** | `messages.loading`, `empty`, `error` | 読み込み中・在庫なし・エラー時 |
| **注意書きの文言** | `notice.text` | 共通で表示する注意書きテキスト |

### 1.2 在庫表示設定に「移す／追加する」と整理しやすいもの

テーマのスキーマにあるが、「全ブロックで揃えたい」なら在庫表示設定にまとめるとよい項目です。

| 項目 | 現在の場所 | 移す理由 |
|------|------------|----------|
| **凡例を表示する** | テーマ（show_legend） | 「凡例を出すか」は表示ルール。全ブロックで統一しやすい。 |
| **注意書きを表示する** | テーマ（show_notice） | 注意書きの ON/OFF も表示ルール。文言はすでにアプリ。 |
| **行内の表示内容** | テーマ（row_content_mode） | マークのみ／マーク＋在庫数／ラベル付き など「1行に何を出すか」は表示ルール。 |
| **リストの区切り文字** | テーマ（list_separator） | 「： 」や「 / 」など、文言に近い。統一すると表記が揃う。 |

※ 凡例の**位置・揃え・色**、注意書きの**位置・揃え・間隔・色**は「見た目」なのでカスタマイザー側に残す想定です。

### 1.3 在庫表示設定としての項目一覧（まとめ後）

| カテゴリ | 項目 |
|----------|------|
| **閾値** | 在庫なしの上限、在庫ありの下限 |
| **在庫マーク** | 在庫あり／残りわずか／在庫なしのマーク文字 |
| **在庫数表示** | 在庫数ラベル、前につける文字、後につける文字 |
| **ステータスラベル** | 在庫あり／残りわずか／在庫なしのラベル文言 |
| **メッセージ** | 読み込み中・在庫なし・エラーのメッセージ |
| **注意書き** | 注意書きの文言、**注意書きを表示する（ON/OFF）** |
| **凡例** | **凡例を表示する（ON/OFF）** |
| **行の表示** | **行内の表示内容**（マークのみ／マーク＋在庫数／マーク＋在庫数＋ラベル／在庫数のみ／在庫数＋ラベル）、**リストの区切り文字** |

※ 太字はテーマから在庫表示設定へ移す候補。

## 2. カスタマイザー（テーマ）で見て調整するもの

「プレビューを見ながら、ブロックごとに見た目を変えたい」項目。文字サイズ・間隔・色・配置など。

### 2.1 タイトル・ブロック全体

| 項目 | スキーマ id 例 | 説明 |
|------|-----------------|------|
| タイトル文言 | heading_text | ブロックごとに変えたい場合はテーマのまま。共通にしたい場合はアプリへ。 |
| タイトルの揃え | heading_align | 左／中央／右 |
| タイトル下マージン | heading_margin_bottom | 間隔 |
| タイトルの色 | heading_color | 色 |
| ブロック上マージン | margin_top | 間隔 |
| ブロック下マージン | margin_bottom | 間隔 |
| 最大幅 | content_max_width | 幅 |
| ブロック全体の配置 | content_align | 左／中央／右 |

### 2.2 レイアウト・行の見た目

| 項目 | スキーマ id 例 | 説明 |
|------|-----------------|------|
| レイアウト種別 | layout_type | 表／リスト／2行（ここは「見た目」としてテーマで調整） |
| ロケーション名の揃え | location_align | 揃え |
| 在庫表示の揃え | status_align | 揃え |
| 文字サイズ | font_size | 小さめ／標準／大きめ |
| 行の上下余白 | row_padding_y | 間隔 |
| 行の左右余白 | row_padding_x | 間隔 |
| ロケーション列の幅（%） | location_col_width | 表レイアウト時の列幅 |

### 2.3 凡例・注意書きの「見た目」

| 項目 | スキーマ id 例 | 説明 |
|------|-----------------|------|
| 凡例の位置 | legend_position | リストの上／下 |
| 凡例の揃え | legend_align | 左／中央／右 |
| 凡例の色 | legend_color | 色 |
| 注意書きの位置 | notice_position | 上／下 |
| 注意書きの揃え | notice_align | 揃え |
| 注意書きとリストの間隔 | notice_margin | 間隔 |
| 注意書きの色 | notice_color | 色 |

※ 「凡例を表示する」「注意書きを表示する」は在庫表示設定に移す想定。

### 2.4 枠線・色

| 項目 | スキーマ id 例 | 説明 |
|------|-----------------|------|
| 枠線スタイル | border_style | なし／行ごと／外枠＋行ごと |
| 枠線の太さ | border_width | 太さ |
| 枠線の色 | border_color | 色 |
| 行内テキストの色 | status_text_color | 色 |
| 在庫ありマークの色 | in_stock_color | 色 |
| 残りわずかマークの色 | low_stock_color | 色 |
| 在庫なしマークの色 | out_of_stock_color | 色 |

### 2.5 エリア見出し・ボタン・その他

| 項目 | スキーマ id 例 | 説明 |
|------|----------------|------|
| エリア折りたたみの開閉マーク | accordion_icon | 三角／山括弧／プラスマイナス／矢印 |
| エリア見出しの背景色・文字色・サイズ・太さ・余白 | region_heading_* | コンポーネントの見た目 |
| 店舗受け取りボタンの背景色・文字色・角丸・余白・文字サイズ | order_pick_btn_* | コンポーネントの見た目 |
| 追加CSS | custom_css | ブロック単位の上書き |

## 3. まとめ方の一覧（実装時の参照用）

### 在庫表示設定（管理画面）にまとめるもの

- 閾値
- 在庫マークの文字
- 在庫数ラベル・前後の文字
- ステータスラベル
- メッセージ文言
- 注意書きの文言 ＋ **注意書きを表示する**
- **凡例を表示する**
- **行内の表示内容**（マークのみ／マーク＋在庫数 など）
- **リストの区切り文字**

→ 保存先: メタフィールド `location_stock.config`。App Proxy の `config` でスニペットに渡す。

### カスタマイザーで見て調整するもの

- タイトル（文言・揃え・下マージン・色）
- レイアウト種別・ロケーション名の揃え・在庫表示の揃え
- 文字サイズ、ブロック・行の余白、最大幅、ブロック全体の配置
- 凡例の**位置・揃え・色**（表示するかはアプリ）
- 注意書きの**位置・揃え・間隔・色**（表示するか・文言はアプリ）
- 枠線（スタイル・太さ・色）、行内テキスト・マークの色
- エリア見出し・店舗受け取りボタンのデザイン
- エリア折りたたみの開閉マーク、追加CSS

→ 保存先: テーマブロックの `block.settings`（スキーマ）。スニペットには `config` として渡す。

## 4. 境界にあるもの・選択肢

| 項目 | 在庫表示設定にすると | カスタマイザーにすると |
|------|----------------------|--------------------------|
| **タイトル文言** | 全ブロックで同じタイトルにできる | ブロックごとに「この商品だけ別タイトル」にできる |
| **レイアウト種別**（表/リスト/2行） | 全ブロックで同じ表示形式にできる | ページやブロックごとに表・リストを切り替えられる |

- **タイトル文言**: 共通化したいなら在庫表示設定に 1 項目追加。ブロックごとに変えたいならカスタマイザーのみ（現状どおり）。
- **レイアウト種別**: 「表示ルール」として統一するなら在庫表示設定、「見た目」としてブロックごとに変えるならカスタマイザー。上記まとめでは「見た目」としてカスタマイザーに含めています。

必要に応じて「デフォルトはアプリで設定し、ブロックで上書き可能」にする設計もできます。

## 5. 実装時の流れ（案）

1. **在庫表示設定（app.settings.jsx）**
   - 既存のまま閾値・マーク・ラベル・メッセージ・注意書き文言を維持。
   - 必要なら `config` に **show_legend**, **show_notice**, **row_content_mode**, **list_separator** を追加し、loader/action で保存・読み込み。

2. **App Proxy（buildGlobalConfig）**
   - 上記を `config` に含めて返す。スニペットは `appConfig` から読み、テーマの値は「アプリに項目がなければフォールバック」にすると後方互換しやすい。

3. **テーマスキーマ**
   - show_legend, show_notice, row_content_mode, list_separator は**削除**するか、**「アプリで未設定のときだけ使う」**フォールバック用に残す。
   - 残す項目は「カスタマイザーで見て調整するもの」だけにし、スキーマのグループ名も「タイトルとレイアウト」「見た目・余白・色」「エリア見出し・ボタン」「その他」のように整理すると分かりやすい。

4. **スニペット**
   - 凡例・注意書き・行の表示内容・区切り文字は、`appConfig` にあればそれを使い、なければ `block.settings`（s.xxx）を使う処理に統一。

この振り分けで実装を進めると、「在庫表示設定にまとめるもの」と「カスタマイザーで見て調整するもの」がはっきりします。
